# Changelog — quiz

所有版本異動依時間倒序排列。

---

## [1.3.0] — 2026-07-12

### Added
- 面試官模式支援指定領域：`/quiz interview {架構|微服務|spring|java|adr}` 直接鎖定領域
- 收尾評析新增「🎯 職級落點」：判定本回答落在 senior / staff / architect 哪個 bar、往上一級要補什麼，附三層級判準表供評析對照
- 新增模式 B-D（`/quiz interview design`）：多階段 system design 長題模擬——需求澄清（含隱藏需求）→ 容量估算 → 高層設計 → deep dive → 故障與演進，分階段互動、每階段最多追問 1 層，收尾評析含分階段小結；支援自帶題目

### Changed
- 面試官模式未指定領域時，選領域由 30 分鐘區間輪替（`/1800`）改為**每次呼叫秒級隨機**（`timestamp % 5`），連續練習不再鎖同一領域；模式 A 維持 30 分鐘區間不變
- 領域 4（ADR 決策復盤）選 ADR 同步改為秒級隨機
- `/loop` 觸發限定模式 A 的說明擴及模式 B-D

---

## [1.2.0] — 2026-07-12

### Added
- 面試官模式新增第 5 個輪替領域「ADR 決策復盤」：從 `common_KBs/ADRs/` 隨機抽一篇 ADR（排除 index 與模板），把候選人當作決策 owner 挑戰決策動機、trade-off、推翻條件與後果緩解
- 領域 4 專屬規則：出題與追問錨定該 ADR 實際記載的 Context / Considered Options / Consequences，不可虛構選項；挑戰框架可用通用知識補充
- 若候選人觀點比 ADR 記載更完整，提示回寫該 ADR 並詢問是否執行 `/update-kb`

### Changed
- 出題領域輪替由 `% 4` 改為 `% 5`
- 知識來源章節區分：領域 0–3 以通用知識出題；領域 4 錨定所抽 ADR 全文
- 領域 4 收尾評析的延伸閱讀改為引用該 ADR 檔案路徑

---

## [1.1.1] — 2026-07-07

### Changed
- 面試官模式難度定位由「中高階後端（senior）」上調為「高階後端與架構師（senior/staff、architect）」
- 面試官人設同步改為面試高階後端與架構師職位
- 難度基準補充架構師視角考點：系統全局取捨、容量與瓶頸推估、故障域、演進成本

## [1.1.0] — 2026-07-07

### Added
- 面試官模式（`/quiz interview` 或 `/quiz 面試`）：模擬科技大廠（MANGO 級）資深面試官，重視底層原理與 not just do but know why，追根究柢式深度面試
- 四大出題領域依 timestamp 輪替：架構設計、微服務、Spring 底層、Java 底層
- 追問機制：依候選人回答品質最多追問 2 層（機制 → trade-off → 邊界條件），答錯先引導修正而非直接給答案
- 收尾評析：面試官視角評價 + 完整參考答案脈絡 + 加分建議 + tech-research 延伸閱讀（有相關檔案時）

### Changed
- 模式判斷章節：無參數走原 KB 溫習模式（模式 A），帶 `interview` 參數走面試官模式（模式 B）
- 注意事項明確標注 KB 內容限制僅適用模式 A；模式 B 以通用技術知識出題與評析
- 模式 B 難度基準為中高階後端（大廠 senior），模式 A 維持中階
- `/loop` 觸發預設走模式 A，面試官模式僅由使用者明確帶參數啟動

## [1.0.0] — 2026-06-27

### Added
- 從 `knowledge/common_KBs/tech-research/` 隨機抽取主題出題
- 以 Unix timestamp 決定選題索引，每 30 分鐘區間對應不同檔案
- 支援選擇題（4 選項）與簡答題兩種題型，依 timestamp 奇偶數交替
- 面試情境式出題風格（情境描述 → 題目問句）
- 回答後提供解析與來源章節引用
- 固定附跳過選項（S），跳過後顯示正確答案與一句話核心概念
- 支援獨立呼叫（`/quiz`）或由 `/loop 30m /quiz` 定時觸發，兩者互不影響
